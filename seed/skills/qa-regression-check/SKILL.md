# QA Regression Check

Use this skill to review implementation outcomes for regressions.

## Review Focus

1. Confirm success-path behavior and edge-case handling.
2. Validate changed contracts in tests.
3. Verify no backwards-incompatible API or schema drift.
4. Summarize release risk with explicit evidence.
