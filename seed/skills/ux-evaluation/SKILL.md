# UX Evaluation

## Overview
- Apply established UX heuristics to evaluate product flows and interactions.
- Identify friction points, cognitive load issues, and navigation inconsistencies.
- Deliver prioritised recommendations aligned with user goals.

## Prerequisites
- Flows or screens to be evaluated are available.
- Target user persona or use case is defined.

## Instructions
1. Map the user flow being evaluated and identify entry and exit points.
2. Apply Nielsen's 10 usability heuristics to each screen or interaction step.
3. Identify friction points where the user must exert extra cognitive effort.
4. Assess information architecture: labels, groupings, and navigation clarity.
5. Evaluate error states and recovery paths.
6. Prioritise findings by user impact and implementation effort.

## Output Format
- Flow map with annotated friction points.
- Heuristic violations table with severity and rationale.
- Prioritised recommendations with acceptance criteria.

## Common Pitfalls
- Evaluating implementation constraints rather than user experience quality.
- Confusing visual polish with usability.
