## Permission Model Summary

Effective agent capability is the intersection of:

1. Agent profile `allowed_tools` and `tool_policy`.
2. Workflow-level `permissions.allow_tools` / `deny_tools`.
3. Job-level or step-level permission overrides.
4. Special-handler policy inputs, e.g. `mcp_tool_call.inputs.policy.allowed_tools`.
5. Runtime mount availability for external MCP servers/tools.

Use the most explicit, owner-qualified tool name in workflows (`kanban.*` for Kanban-owned tools). Avoid old aliases or unqualified Kanban state helpers.
